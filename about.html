Câu 1 (10 điểm): Sự khác biệt giữa position: absolute và position: relative

Trong CSS, thuộc tính position dùng để xác định cách phần tử được định vị trên trang web.

1. position: relative

  • Phần tử vẫn giữ vị trí ban đầu trong luồng (flow) của trang.
  • Có thể di chuyển bằng các thuộc tính top, left, right, bottom.
  • Khoảng trống ban đầu của phần tử vẫn được giữ lại.

Ví dụ:

<div class="box">Box</div>

.box {
  position: relative;
  top: 20px;
  left: 30px;
  background: lightblue;
}

Kết quả: Box dịch xuống 20px và sang phải 30px. Vị trí cũ của nó vẫn được giữ trong bố cục.

2. position: absolute

  • Phần tử bị loại khỏi luồng của trang.
  • Được định vị theo phần tử cha gần nhất có position khác static.
  • Nếu không có cha phù hợp thì sẽ căn theo cửa sổ trình duyệt.

Ví dụ:

<div class="container">
  <div class="box">Box</div>
</div>

.container {
  position: relative;
  width: 300px;
  height: 200px;
}

.box {
  position: absolute;
  top: 20px;
  right: 10px;
}

Kết quả: Box nằm ở góc trên bên phải của .container. Không chiếm không gian trong bố cục.

Ứng dụng thực tế

position: relative

  • Làm khung chứa cho phần tử absolute.
  • Điều chỉnh vị trí nhỏ của icon.
  • Thiết kế layout responsive.

position: absolute

  • Popup.
  • Tooltip.
  • Menu thả xuống.
  • Nút nổi.
  • Badge thông báo.

Ảnh hưởng đến Responsive

  • relative giúp giao diện linh hoạt hơn trên nhiều kích thước màn hình.
  • absolute nếu dùng quá nhiều với giá trị pixel cố định sẽ dễ bị lệch trên điện thoại hoặc tablet.
  • Khi thiết kế responsive nên kết hợp Flexbox hoặc Grid thay vì lạm dụng Absolute.

Ảnh hưởng khi Deploy Website

Khi deploy lên Hosting hoặc GitHub Pages:

  • CSS hoạt động giống nhau.
  • Nếu layout sử dụng absolute không hợp lý thì trên các thiết bị khác nhau giao diện sẽ dễ vỡ.
  • Cần kiểm tra Responsive bằng Chrome DevTools trước khi deploy.

Câu 2 (10 điểm): Quy trình đăng ký Domain và Hosting

Một website JavaScript thông thường cần các bước:

Bước 1: Đăng ký tên miền (Domain)

Ví dụ: abc.com, mywebsite.vn

Bước 2: Thuê Hosting

Hosting là nơi lưu: HTML, CSS, JavaScript, Hình ảnh.

Bước 3: Upload Source Code

Có thể upload bằng: File Manager, FTP, Git.

Bước 4: Trỏ Domain về Hosting

Thông qua DNS.

Vai trò của DNS

DNS (Domain Name System) giúp chuyển www.example.com thành 192.xxx.xxx.xxx để trình duyệt tìm đúng máy chủ chứa website.

Nếu cấu hình DNS sai:

  • Không truy cập được website.
  • Website báo lỗi.
  • Có thể truy cập nhầm máy chủ.

Hai công cụ Deploy miễn phí

1. GitHub Pages — Ưu điểm:

  • Miễn phí.
  • Dễ sử dụng.
  • Tự động deploy từ GitHub.
  • Phù hợp website tĩnh.

2. Netlify — Ưu điểm:

  • Deploy tự động.
  • Hỗ trợ HTTPS.
  • Custom Domain.
  • Tốc độ nhanh.

Ngoài ra còn có: Vercel, Firebase Hosting.

Rủi ro bảo mật

Nếu cấu hình sai có thể xảy ra:

  • Không bật HTTPS.
  • Lộ source code.
  • Sai quyền truy cập thư mục.
  • DNS bị giả mạo (DNS Spoofing).
  • Domain bị chiếm quyền nếu không bật bảo vệ.

Do đó cần:

  • Sử dụng HTTPS.
  • Thiết lập quyền truy cập đúng.
  • Cấu hình DNS chính xác.
  • Bật xác thực hai lớp cho tài khoản quản lý.

Câu 3 (10 điểm): So sánh var, let và const

Tiêu chí var let const
ScopeFunctionBlockBlock
HoistingCóCó nhưng chưa khởi tạoCó nhưng chưa khởi tạo
Khai báo lạiCóKhôngKhông
Gán lạiCóCóKhông

Ví dụ lỗi thường gặp

console.log(a);

let a = 10;

Kết quả: ReferenceError — Do let được hoisting nhưng chưa được khởi tạo.

Ví dụ khác:

const PI = 3.14;

PI = 3;

Kết quả: TypeError — Vì const không cho phép gán lại.

Ví dụ về var:

if (true) {
  var x = 5;
}

console.log(x);

Kết quả: 5 — Do var không có Block Scope.

Debug bằng Chrome DevTools

Các bước:

  • Mở Chrome.
  • Nhấn F12.
  • Chọn tab Sources.
  • Đặt Breakpoint.
  • Chạy chương trình.
  • Quan sát giá trị biến trong Scope.
  • Kiểm tra lỗi ở tab Console.

DevTools giúp xác định: ReferenceError, TypeError, Giá trị biến, Stack Trace, Quá trình thực thi chương trình.

Câu 4 (10 điểm): Khái niệm Event Delegation

Event Delegation là kỹ thuật gắn một Event Listener lên phần tử cha để xử lý sự kiện của các phần tử con thông qua cơ chế Event Bubbling.

Thay vì gắn nhiều Listener cho từng phần tử, chỉ cần một Listener ở phần tử cha.

Ví dụ

<ul id="list">
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

document.getElementById("list").addEventListener("click", function (e) {
  if (e.target.tagName === "LI") {
    alert(e.target.innerText);
  }
});

Khi click vào bất kỳ thẻ <li> nào đều hiển thị nội dung của nó.

Ứng dụng với danh sách động

let li = document.createElement("li");
li.innerText = "React";
document.getElementById("list").appendChild(li);

Dù phần tử được thêm sau khi trang đã tải, Event Delegation vẫn xử lý được mà không cần gắn thêm Event Listener.

Lợi ích

  • Giảm số lượng Event Listener.
  • Tiết kiệm bộ nhớ.
  • Tăng hiệu suất website.
  • Dễ bảo trì mã nguồn.
  • Hoạt động tốt với các phần tử được tạo động bằng JavaScript.
  • Dễ kết hợp với HTML và CSS để xây dựng giao diện tương tác.

Kết luận

  • position: relative và absolute giúp định vị phần tử, nhưng cần sử dụng hợp lý để đảm bảo giao diện responsive.
  • Domain, Hosting và DNS là ba thành phần quan trọng để đưa website lên Internet; GitHub Pages và Netlify là hai giải pháp deploy miễn phí hiệu quả.
  • let, const và var khác nhau về phạm vi hoạt động (scope) và cơ chế hoisting; Chrome DevTools hỗ trợ phát hiện và sửa lỗi nhanh chóng.
  • Event Delegation giúp xử lý sự kiện hiệu quả, đặc biệt với các danh sách động, góp phần tối ưu hiệu suất và khả năng bảo trì của website.
